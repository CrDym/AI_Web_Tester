# ROADMAP

本文件描述 AI Web Tester 的后续演进路线：从“可用的自愈自动化工具”走向“可规模化的团队测试平台”。  
按优先级分为 P0（可用性补齐）→ P1（自愈可信）→ P2（录制体验）→ P3（平台化/CI）。

## 当前状态（已具备）
- Web 控制台：用例大厅、编辑器、运行、回放、截图缩略条、自愈记录展示
- 用例创建：Chrome 插件录制 + NL2Case 自然语言生成
- 自愈闭环：自愈事件上报 + 手工“批准更新”写回用例
- 数据存储：用例 JSON（tests/recorded_cases）、运行目录（tests/run_history）

## P0：可用性补齐（让日常回归“顺手”）
**目标**：让用户不需要理解内部细节，就能稳定创建/运行/回放/清理用例。

- **用例管理**
  - [x] 标签系统：用例增加自定义标签（Tags），支持左侧大厅按标签快速筛选和高亮展示
  - [ ] 用例参数化与数据驱动：支持定义 `${username}`, `${password}` 变量，并在套件级别绑定多组数据集（如 Excel/CSV）实现循环执行
  - [ ] 文件夹与树状结构：用例按业务模块层级归类展示
  - [ ] 批量操作：批量删除、批量改名、批量打标签
  - [ ] 用例复制、导入/导出（JSON 打包）
- **套件（Suite）与测试计划（P0 落地：Suite + Suite Run）**
  - [x] Suite：多个 case_id 的有序集合（套件内不允许重复），可绑定统一 env_id
  - [x] Suite Run：一键顺序执行所有用例，失败继续跑，生成聚合报告并可跳转到单个 run 回放
  - [x] 前置用例（setup_case）：可选，套件运行时优先执行，用于登录等准备动作，并通过 cookie(storageState) 共享给后续用例
  - [x] 数据落盘：`tests/suites/` 与 `tests/suite_history/`
- **运行体验**
  - [ ] 失败原因结构化：定位失败/遮挡/跨域 iframe/strict mode/超时分类
  - [ ] 一键导出单次运行产物（meta.json + logs + screenshots zip）
  - [x] Token 消耗统计：对所有大模型调用展示 token（运行记录、套件汇总、自愈事件、修复建议、配置测试、NL2Case 生成）
- **配置与安全**
  - [x] Web 端可视化大模型配置（API Base、API Key、模型名称）及环境列表（Base URL）管理
  - [ ] 多模型/多 Key 细粒度配置（每个环境/每个用例可选模型）
  - [x] Key 本地安全存储策略（落盘到 `.env` 防止硬编码泄露）

## P1：自愈可信化（从“能跑通”到“可审计、可回归”）
**目标**：让自愈结果可解释、可验证、可回滚、可复现，减少“看起来修了但其实不稳”的情况。

- **自愈审计 (Heal Audit) 与代码回写升级**
  - [x] heal_events 时间线与双屏对比：直观展示“自愈前截图+旧 DOM” vs “自愈后截图+新 DOM”，提供类似 Code Review 的手工审核界面
  - [ ] 基于 AST 的稳定回写：抛弃单纯字符串查找，使用 `ast` 模块精确定位 Python 节点，实现 100% 安全的自愈代码落盘
  - [ ] 一键回滚：自愈错误时支持秒切回历史 selector 版本
- **智能断言推荐**
  - [ ] 大模型视觉感知断言：AI 根据截图变化（如弹出绿色 Toast 或状态跳转），自动推荐 `assert visible` 或 `assert url`，一键应用
- **用例修复建议**
  - [x] 失败时生成 AI 修复建议：分析失败 step/日志/自愈候选，输出根因与可执行改法，并支持一键应用到用例 steps
- **选择器策略与评分**
  - [ ] 生成选择器时优先 data-testid / role / aria，其次 text/结构路径
  - [ ] 对候选 selector 计算稳定性评分（唯一性、文本波动风险、层级深度）
  - [ ] 低分 selector 默认不自动回写，只进入“建议列表”
- **自愈验证**
  - [ ] 回写前做二次验证：新 selector 是否唯一、是否可点击/可输入
  - [ ] 支持“锁定 selector”（禁止被后续自愈覆盖）
  - [ ] Prompt 提示词外部化：抽离并提供可视化入口，允许业务线根据自身 UI 库定制化 AI 的寻素人设与规则

## P2：录制插件升级（把“录制”变成“高质量用例生产线”）
**目标**：录制结果默认就“可跑、可读、可维护”，尽量少让用户回到编辑器手工整理。

- **事件建模更精确**
  - [ ] 输入优化：合并 input + blur，连续输入更新同一指令；支持 `textarea`、富文本 (`contenteditable`)
  - [ ] 选项交互：针对 checkbox、radio 和 select 产生专门动作（而非统一当成 click 处理）
  - [ ] SPA 路由补获：通过监听 `history.pushState`/`replaceState` 自动为单页应用（React/Vue）补充 `wait` 和 URL 断言
  - [ ] 悬停（Hover）与拖拽捕捉
- **录制后智能整理**
  - [ ] 自动去重与合并：消除无意义和重复的 click 动作
  - [ ] 隐式等待推荐：针对加载、导航和动画效果，给出隐式等待的参数推荐
- **意图（Intent）增强**
  - [ ] 提取父级上下文：录制时提取出“右侧设置面板内的保存按钮”等强语义
  
## P3：平台化与工程化（团队协作 + CI）
**目标**：支持团队规模、代码仓库集成、持续回归。

- **CI/CD 与自动化流水线接入**
  - [ ] Webhook 触发机制：提供标准 API 允许 Jenkins/GitLab CI 部署完毕后自动化回调触发套件
  - [ ] 定时任务 (Cron)：允许在 Web 界面配置定时调度策略，支持按天/周自动回归主干流程
  - [ ] 无 UI 执行与跨平台：集成 JUnit/Allure 测试报告并支持飞书、钉钉消息推送
- **多用户与权限**
  - [ ] 项目隔离与空间管控：按业务线划分用例隔离，设定查看/编辑权限
  - [ ] 并发控制池与配额限制
- **版本管理**
  - [ ] 用例变更可溯源：记录并版本化每一次用户/AI 的修改，支持细粒度回退

## 建议的下一步（推荐优先做）
- **核心体验与 UX 提升 (P0 核心收尾)**：用例参数化与数据驱动（解决多数据用例运行痛点）
- **自愈审计大盘 (Heal Audit)**：给团队直观展现“AI 改了哪里、为什么改”，提升白盒信任度
- **智能断言 (AI Assert)**：在运行/自愈时借助大模型直接补充测试校验点
