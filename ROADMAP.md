# ROADMAP

本文件描述 SOLO AI（AI Web Tester）的后续演进路线，目标以“测试团队内部日常回归使用”为主：可用、稳定、可审计、可维护。  
不做过度平台化：默认单实例、单团队使用，优先把核心回归闭环打磨到足够顺手。

## 范围与原则
- 主要服务对象：测试/QA/SDET 团队内部
- 部署形态：本地或内网自托管（单实例）
- 优先级：可用性与稳定性 > 可审计与回滚 > 体验增强
- 非目标（暂不做）：复杂 RBAC/多租户/计费配额/大规模调度平台

## 当前状态（已具备）
- Web 控制台：侧边栏（用例/套件/历史 Tab）、编辑器、运行/回放、截图缩略条、自愈记录展示
- 鉴权：登录/初始化账号（Token），未登录不可访问任何 API
- 主题：浅色/深色主题切换（全局样式已覆盖大部分常用组件）
- 用例创建：Chrome 插件录制 + 自动同步到控制台 + NL2Case 自然语言生成
- 用例参数化：用例级 `variables`、CSV/JSON `dataset`、`${变量名}` 替换、内置日期变量（如 `${today_ymd}`）
- 录制插件：统一 SOLO AI 图标与控制台风格，支持 click/input/select/checkbox/radio/contenteditable/SPA 路由补获，停止录制后自动同步
- 自愈闭环：自愈事件上报 + 手工“批准更新”写回用例
- 运行历史：失败原因筛选、批量选择与批量删除运行记录
- Token 消耗统计：运行记录、套件汇总、自愈事件、修复建议、配置测试、NL2Case 生成等全链路展示
- 数据存储：SQLite（tests/tester.db）、运行目录（tests/run_history）、套件运行目录（tests/suite_history）

## 近期：可用性补齐（让日常回归“顺手”）
**目标**：稳定创建/运行/回放/清理；减少重复操作与手工排错。

- **用例管理**
  - [x] 标签系统：用例增加自定义标签（Tags），支持左侧大厅按标签快速筛选和高亮展示
  - [x] 用例参数化与数据驱动 MVP：用例级变量替换（`${username}` 等）、CSV/JSON 数据集、内置日期变量（`${today_ymd}` 等）
  - [ ] 数据驱动增强：套件级变量/数据集、运行结果按数据行聚合展示
  - [ ] 文件夹/分组：用例按业务模块归类（树状或分组列表）
  - [ ] 批量操作（用例级）：批量删除、批量改名、批量打标签
  - [ ] 用例复制、导入/导出（JSON 打包）
  - [ ] 文案中文化收尾：控制台残留英文文案清理，统一术语与标题
- **套件与回归组织**
  - [x] Suite：多个 case_id 的有序集合（套件内不允许重复），可绑定统一 env_id
  - [x] Suite Run：一键顺序执行所有用例，失败继续跑，生成聚合报告并可跳转到单个 run 回放
  - [x] 前置用例（setup_case）：可选，套件运行时优先执行，用于登录等准备动作，并通过 cookie(storageState) 共享给后续用例
  - [x] 数据落盘：`tests/suites/` 与 `tests/suite_history/`
  - [ ] 回归选择与复跑：支持按“失败用例/标记用例/标签”快速生成临时套件并复跑
- **运行体验**
  - [ ] 失败原因结构化：定位失败/遮挡/跨域 iframe/strict mode/超时等分类与建议动作
  - [ ] 一键导出单次运行产物（meta.json + logs + screenshots zip）
  - [x] 运行历史批量操作：批量选择与批量删除运行记录（必要时逐条删除兜底）
  - [ ] 清理策略：按天/数量保留 run_history（本地清理按钮 + 后端清理接口）
- **配置与安全**
  - [x] Web 端可视化大模型配置（API Base、API Key、模型名称）及环境列表（Base URL）管理
  - [ ] 多模型/多 Key（轻量）：优先支持“按环境选择模型/Key”
  - [x] Key 本地安全存储策略（落盘到 `.env` 防止硬编码泄露）
  - [x] 基础鉴权（单用户）：登录/初始化账号（Token），未登录不可访问数据接口

## 中期：自愈可信化（从“能跑通”到“可审计、可回滚”）
**目标**：把自愈变成“可控的变更”，减少误修风险。

- **自愈审计 (Heal Audit) 与回写升级**
  - [x] heal_events 时间线与双屏对比：直观展示“自愈前截图+旧 DOM” vs “自愈后截图+新 DOM”，提供类似 Code Review 的手工审核界面
  - [ ] 基于 AST 的稳定回写：使用 `ast` 精确定位 Python 节点，避免字符串替换误写
  - [ ] 一键回滚：自愈错误时可切回历史 selector 版本
- **断言能力**
  - [ ] 内置断言模板：URL/可见/文本/状态等常用断言
  - [ ] AI 建议断言：仅做推荐与填参，一键应用到步骤
- **选择器策略与验证**
  - [ ] 生成选择器时优先 data-testid / role / aria，其次 text/结构路径
  - [ ] 选择器稳定性评分（唯一性、文本波动风险、层级深度）
  - [ ] 回写前二次验证：唯一性 + 可点击/可输入
  - [ ] 锁定 selector：禁止被后续自愈覆盖
- **用例修复建议**
  - [x] 失败时生成 AI 修复建议：分析失败 step/日志/自愈候选，输出根因与可执行改法，并支持一键应用到用例 steps

## 远期：可选增强（录制质量与轻量 CI）
**目标**：提升资产生产效率与回归接入工程链路，但保持“内部工具”体量。

- **录制插件（可选）**
  - [x] 输入优化：连续 input/change 合并为同一步；支持 `textarea`、富文本 (`contenteditable`)
  - [x] 选项交互：checkbox/radio/select 产生专门动作（`set_checked` / `select_option`）
  - [x] SPA 路由补获：监听 `history.pushState`/`replaceState` 补充 `wait` 与 URL 断言
  - [x] 停止录制后自动同步到控制台，并提供失败降级下载 JSON
  - [x] 录制器 UI/图标与 Web 控制台保持一致
  - [ ] 悬停与拖拽捕捉
  - [ ] 录制后整理：去重/合并、隐式等待推荐、意图增强（父级上下文）
- **轻量 CI（可选）**
  - [ ] Headless 执行：提供 CLI/脚本入口运行套件（不依赖 UI）
  - [ ] 报告导出：JUnit（必选）+ HTML 简报（可选）；失败时打包产物供排查
  - [ ] 通知（可选）：飞书/钉钉 Webhook（仅发送链接与摘要）

## 建议的下一步（推荐优先做）
1. **失败原因结构化 + 运行产物导出**：把定位失败、遮挡、iframe、strict mode、超时等分类展示，并支持一键导出 meta/logs/screenshots，优先降低排障成本。
2. **录制后整理**：停止录制后自动去重/合并、推荐等待、补强意图，让插件产出的用例更接近可直接回归。
3. **用例分组与批量操作**：按业务模块管理用例，补齐批量删除/改名/打标签，提升测试团队维护大量用例的效率。
4. **数据驱动增强**：在现有 case 级 variables/dataset 基础上，补套件级变量、数据行维度运行结果聚合。
5. **AST 回写 + 一键回滚**：把自愈 selector 回写做成更安全、可回退的变更。
