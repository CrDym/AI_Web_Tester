# ROADMAP

本文件描述 SOLO AI（AI Web Tester）的后续演进路线，目标以“测试团队内部日常回归使用”为主：可用、稳定、可审计、可维护。  
不做过度平台化：默认单实例、单团队使用，优先把核心回归闭环打磨到足够顺手。

## 范围与原则
- 主要服务对象：测试/QA/SDET 团队内部
- 部署形态：本地或内网自托管（单实例）
- 优先级：可用性与稳定性 > 可审计与回滚 > 体验增强
- 非目标（暂不做）：复杂 RBAC/多租户/计费配额/大规模调度平台

## 当前状态（已具备）
- Web 控制台：侧边栏（用例/套件/历史 Tab）、编辑器、运行/回放、截图缩略条、自愈记录展示
- 鉴权：登录/初始化账号（Token），未登录不可访问任何 API
- 主题：浅色/深色主题切换，常用组件已覆盖深色模式
- 用例创建：Chrome 插件录制 + 自动同步到控制台 + NL2Case 自然语言生成
- 用例管理：标签、分组、搜索、批量移动分组、稳定系统 ID（重命名不影响历史和套件引用）
- 用例参数化：用例级 `variables`、CSV/JSON `dataset`、`${变量名}` 替换、内置日期变量（如 `${today_ymd}`）
- 录制插件：统一 SOLO AI 图标与控制台风格，支持 click/input/select/checkbox/radio/contenteditable/SPA 路由补获，停止录制后自动同步
- 自愈闭环：自愈事件上报 + 手工“批准更新”写回用例 + AI 修复建议一键应用
- 运行历史：全量历史、按用例历史、失败原因筛选、批量选择、软删除到 `_deleted`
- 运行详情：运行诊断摘要、失败原因卡片、日志分类过滤、截图回放、自愈记录、Token 明细
- Token 消耗统计：运行记录、套件汇总、自愈事件、修复建议、配置测试、NL2Case 生成等全链路展示
- 数据存储：SQLite（tests/tester.db）、运行目录（tests/run_history）、套件运行目录（tests/suite_history）

## 近期：可用性与排障效率
**目标**：让“运行失败后定位原因、修复用例、再次回归”的闭环更短。

- **运行详情诊断**
  - [x] 运行诊断摘要：状态、耗时、日志数、截图数、自愈数、Token 消耗
  - [x] 失败诊断卡片：失败类型、失败步骤、错误摘要、建议动作
  - [x] 日志分类过滤：全部 / 错误 / 警告 / AI / 成功
  - [ ] 日志与截图时间线联动：点击截图时定位相邻日志；点击错误日志时定位相邻截图
  - [ ] 一键导出单次运行产物：meta.json + logs + screenshots zip
- **用例编辑器体验**
  - [ ] 步骤折叠：默认只展示步骤摘要，点击后展开编辑
  - [ ] 步骤复制/禁用：支持复制步骤、临时禁用步骤，运行时跳过禁用步骤
  - [ ] Selector 辅助提示：标识 CSS/XPath/Text/Role/TestID，提示更稳定的选择器写法
  - [ ] 变量提示：输入框中提示 `{{username}}`、`${today_ymd}`、环境变量等可用变量
- **录制器归档流程**
  - [ ] 录制完成后弹出归档面板：名称、分组、标签、环境、是否整理步骤
  - [ ] 录制后整理：去重/合并输入步骤、补充等待、生成步骤意图摘要
- **运行历史与清理**
  - [x] 运行历史软删除：删除后移动到 `tests/run_history/_deleted`
  - [ ] 清理策略：按天/数量保留 run_history（本地清理按钮 + 后端清理接口）
  - [ ] 恢复入口：从 `_deleted` 恢复误删运行记录

## 中期：套件与回归组织
**目标**：从“跑单条用例”升级为“稳定组织业务流程回归”。

- **套件运行能力**
  - [x] Suite：多个 case_id 的有序集合（套件内不允许重复），可绑定统一 env_id
  - [x] Suite Run：一键顺序执行所有用例，失败继续跑，生成聚合报告并可跳转到单个 run 回放
  - [x] 前置用例（setup_case）：运行套件前优先执行，并通过 cookie(storageState) 共享给后续用例
  - [ ] 失败策略：失败继续 / 失败即停 / 只重试失败用例
  - [ ] 回归选择与复跑：支持按“失败用例/标记用例/标签”快速生成临时套件并复跑
  - [ ] 套件级变量与数据集：套件统一注入变量，运行结果按数据行聚合
- **运行队列**
  - [ ] 后端运行队列：等待中、运行中、成功、失败、取消
  - [ ] 前端运行任务面板：展示当前运行中和等待中的任务
  - [ ] 取消运行：支持取消单条运行或套件运行
  - [ ] 防重复运行：同一用例运行中时提示是否继续新开一次

## 中期：自愈可信化
**目标**：把自愈变成“可控的变更”，减少误修风险。

- **自愈审计与回写升级**
  - [x] heal_events 时间线与双屏对比：展示“自愈前截图+旧 DOM” vs “自愈后截图+新 DOM”
  - [x] 手工批准更新：用户确认后再写回用例
  - [ ] 基于 AST 的稳定回写：使用 `ast` 精确定位 Python 节点，避免字符串替换误写
  - [ ] 一键回滚：自愈错误时可切回历史 selector 版本
  - [ ] 锁定 selector：禁止被后续自愈覆盖
- **断言能力**
  - [ ] 内置断言模板：URL/可见/文本/状态等常用断言
  - [ ] AI 建议断言：仅做推荐与填参，一键应用到步骤
- **选择器策略与验证**
  - [ ] 生成选择器时优先 data-testid / role / aria，其次 text/结构路径
  - [ ] 选择器稳定性评分：唯一性、文本波动风险、层级深度
  - [ ] 回写前二次验证：唯一性 + 可点击/可输入

## 远期：轻量工程化与 CI
**目标**：提升回归接入工程链路，但保持“内部工具”体量。

- **轻量 CI**
  - [ ] Headless 执行：提供 CLI/脚本入口运行套件（不依赖 UI）
  - [ ] 报告导出：JUnit（必选）+ HTML 简报（可选）；失败时打包产物供排查
  - [ ] 通知（可选）：飞书/钉钉 Webhook（仅发送链接与摘要）
- **数据一致性保障**
  - [x] 稳定 ID：case/suite/env 使用系统 ID，名称只作为展示字段
  - [ ] 迁移脚本：整理稳定 ID 和历史引用修复脚本，支持老数据升级
  - [ ] 数据一致性测试：重命名 case/suite/env 后历史、套件、环境引用不断链

## 建议的下一步（推荐优先做）
1. **日志与截图时间线联动**：在诊断面板基础上，把失败日志、截图和步骤串起来，进一步降低排障成本。
2. **用例步骤折叠 + 复制/禁用**：提升长用例维护体验，减少编辑误操作。
3. **录制器归档流程**：录制结束时直接设置名称、分组、标签和环境，减少后续整理成本。
4. **运行队列与取消能力**：为套件运行和批量回归打基础，避免多浏览器并发混乱。
5. **数据一致性测试与迁移脚本**：把稳定 ID 策略固化为可验证的工程保障。
