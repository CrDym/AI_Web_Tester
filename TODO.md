# 📝 AI Web Tester 进阶开发计划

## 🚀 P0：核心稳定性与功能完善
- [ ] **跨页面的 Context 传递支持**：目前单步操作是独立的，如果前一个 intent 打开了新标签页，需要有更优雅的机制捕获新页面的 Playwright page 对象。
- [ ] **多模态能力的进阶优化**：红框标注目前只在部分组件上生效，针对 Canvas、复杂 iframe 及高度定制化组件的视觉截取和定位需要强化。
- [ ] **多模型并发仲裁机制**：对于极为复杂的页面，支持同时呼叫不同模型 (如 GPT-4o 和 Claude 3.5)，取置信度高的动作执行，降低单一模型“幻觉”概率。
- [ ] **重试与降级策略**：增强 DOM 获取阶段的稳健性，在页面发生大规模异步渲染导致 DOM 解析失败时，支持重试或 fallback。

## 📊 P1：数据管理与生态融合
- [ ] **测试数据驱动 (DDT) 的支持**：让 `AITesterAgent.step()` 能够接受外部的数据驱动列表 (如 Excel、CSV)，并在多组数据间复用同一套执行缓存。
- [ ] **与主流 CI/CD 的深度集成**：提供一键生成 GitHub Actions, GitLab CI 或 Jenkins 管道配置的命令。
- [ ] **Allure Report 深度集成**：除了现在的轻量级 HTML 报告外，接入 Allure，把 Agent 每一步思考的 Prompt、截图、耗时全景记录进去。

## 🧠 P2：更聪明的 AI 特性
- [ ] **全局业务知识库 (RAG) 接入**：对于像后台管理端这种复杂系统，支持预先喂入系统的“使用手册”或“业务逻辑树”，让 AI 不仅能“看懂页面”，还能“懂业务规则”。
- [ ] **自我进化模型 (Fine-tuning 管道)**：记录每一次由 `SelfHealer` 修复的错误选择器数据，定期输出数据集，供团队微调本地小模型，降低对商业大模型的依赖和成本。
- [ ] **移动端 H5 及 Appium 扩展**：将驱动层从单纯的 Playwright 扩展，支持通过 Appium 接管真实的 iOS/Android 设备。
