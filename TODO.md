# 📝 AI Web Tester 近期开发计划

这里记录了框架在近期迭代中需要优化和完善的务实功能：

## 🚀 新架构 (B/S Web 控制台) 增强
- [x] **断言 (Assert) 步骤**：编辑器支持断言步骤（文本包含、URL 包含、元素可见性等），后端脚本生成器输出对应断言代码。
- [x] **多环境管理 (Dev/QA/Prod)**：支持配置不同的环境 `base_url`、账号信息，运行时可一键切换环境注入。
- [x] **用例编辑器增强**：支持拖拽排序步骤、批量复制、临时禁用某一步，以及按名称或标签搜索用例。
- [x] **自愈审核页 (Healing Review)**：展示每次自愈前后的 selector 变更与截图对比，支持用户一键“批准/回滚/锁定该选择器”。

## 🛠️ 稳定性与细节优化
- [x] **多标签页 (New Tab) 支持**：目前操作仅限于当前页面，如果点击某个链接打开了新标签页，Agent 还没法自动切过去。需要增加机制捕获并切换 Playwright 的 `page` 对象。
- [x] **iframe 穿透**：很多企业后台的表单、上传组件嵌套在 iframe 里，当前的 DOM 提取脚本还没法有效穿透跨域或多层 iframe，这块必须补上。
- [x] **悬浮层/弹窗遮挡处理优化**：有时候大模型想点的元素被一个半透明的全局 Mask 或者 Toast 提示挡住了（导致 Playwright 报 `intercepts pointer events`），需要让 Agent 能更聪明地识别遮挡物并尝试关闭它。
- [x] **滚动查找优化 (Scroll into view)**：现在的 `scroll` 动作比较粗暴（默认往下滑），遇到需要“在局部长列表里滚动查找特定元素”时效率不高。
- [x] **自然语言生成测试用例 (NL2Case)**：支持通过输入自然语言测试步骤，由大模型自动拆解为标准测试用例结构，并结合底层自愈引擎在运行时动态寻找真实元素。
- [x] **运行回放图片展示修复**：回放模式下大图偶发不居中/不完整显示，需要重新梳理右侧图片容器的 flex 高度计算与 `object-fit` 策略，并做跨分辨率验证（含缩略条存在时）。

## 📊 效率与易用性
- [ ] **测试数据参数化 (Data-Driven)**：让用例步骤可以支持传入变量，使得同一套“意图脚本”能够循环跑不同的测试账号和数据。
- [ ] **定时任务与 CI/CD 接入**：Web 后端增加定时调度能力（Cron），并提供 Webhook 触发接口供 GitLab/Jenkins/GitHub Actions 接入流水线。

## ⚙️ 底层架构
- [ ] **Playwright 驱动层抽象**：现在的 `PlaywrightDriver` 还有些操作写死在里面，考虑进一步抽象接口，方便后续大家根据自己的业务需要扩展（比如增加双击、拖拽、长按等复杂交互）。
- [ ] **Prompt 模板外置**：目前 `AITesterAgent` 和 `SelfHealer` 里的系统提示词是写在代码里的，未来可以抽离，方便不同业务团队自己魔改大模型的人设和规则。
