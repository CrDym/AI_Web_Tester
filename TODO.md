# 📝 AI Web Tester 近期开发计划

这里记录了框架在近期迭代中需要优化和完善的务实功能：

## 🛠️ 稳定性与细节优化
- [ ] **多标签页 (New Tab) 支持**：目前操作仅限于当前页面，如果点击某个链接打开了新标签页，Agent 还没法自动切过去。需要增加机制捕获并切换 Playwright 的 `page` 对象。
- [ ] **iframe 穿透**：很多企业后台的表单、上传组件嵌套在 iframe 里，当前的 DOM 提取脚本还没法有效穿透跨域或多层 iframe，这块必须补上。
- [ ] **悬浮层/弹窗遮挡处理优化**：有时候大模型想点的元素被一个半透明的全局 Mask 或者 Toast 提示挡住了（导致 Playwright 报 `intercepts pointer events`），需要让 Agent 能更聪明地识别遮挡物并尝试关闭它。
- [ ] **滚动查找优化 (Scroll into view)**：现在的 `scroll` 动作比较粗暴（默认往下滑），遇到需要“在局部长列表里滚动查找特定元素”时效率不高。

## 📊 效率与易用性
- [ ] **测试数据参数化 (Data-Driven)**：让 `agent.step()` 可以支持传入字典变量，使得同一套“意图脚本”能够循环跑不同的测试账号和数据，而不需要重新让大模型分析页面。
- [ ] **更清晰的失败报错**：当大模型连续几次都找不到目标元素导致意图失败时，直接把最后一步的带红框截图高亮显示在 HTML 报告里，让排查一目了然。
- [ ] **CI/CD 接入示例**：在 `examples/` 里补充一个 GitHub Actions 或 GitLab CI 的 yml 配置文件，演示怎么把无头模式的测试无缝塞进流水线里跑。

## ⚙️ 底层架构
- [ ] **Playwright 驱动层抽象**：现在的 `PlaywrightDriver` 还有些操作写死在里面，考虑进一步抽象接口，方便后续大家根据自己的业务需要扩展（比如增加双击、拖拽、长按等复杂交互）。
- [ ] **Prompt 模板外置**：目前 `AITesterAgent` 里的系统提示词是写在代码里的，未来可以抽离到单独的 `.txt` 或 `yaml` 里，方便不同业务团队自己魔改大模型的人设和规则。
